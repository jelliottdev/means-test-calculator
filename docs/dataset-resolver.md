# Dataset Resolver

The project now includes a dataset resolver abstraction so the engine can run from resolved dataset bundles instead of importing constants directly.

## New modules

- `src/datasets/types.ts`
- `src/datasets/embedded.ts`
- `src/datasets/resolver.ts`
- `src/datasets/index.ts`
- `src/engine/v2/meansTestFromBundle.ts`
- `src/engine/v2/runner.ts`

## Why this matters

This is the seam needed to remove hardcoded runtime behavior.

The calculation flow becomes:

1. Resolve datasets for filing date
2. Normalize case facts
3. Run deterministic means test from bundle
4. Return result + audit metadata

## Current status

The default resolver now returns an embedded bundle assembled from the checked-in versioned artifact JSON manifests, so the runtime path matches the artifact/publication pipeline for supported filing dates.

## Target status

Keep the embedded resolver for static/browser builds, and pair it with file-backed or database-backed resolvers that load the same versioned artifacts generated by the official data ingestion pipeline.

## Example

```ts
import { runMeansTestResolved } from "./src/engine/v2/runner";
import { normalizeLegacyInput } from "./src/engine/v2";

const result = await runMeansTestResolved(normalizeLegacyInput(legacyInput));
```

## Existing file-backed layout

The repo already supports a file-backed resolver using a structure like:

```text
/data/means-test/
  /2025-11-01/
    median-income.json
    national-standards.json
    housing.json
    transportation.json
    thresholds.json
```
