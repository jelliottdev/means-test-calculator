# Dataset Resolver

The project now includes a dataset resolver abstraction so the engine can run from resolved dataset bundles instead of importing constants directly.

## New modules

- `src/datasets/types.ts`
- `src/datasets/embedded.ts`
- `src/datasets/resolver.ts`
- `src/datasets/index.ts`
- `src/engine/v2/meansTestFromBundle.ts`
- `src/engine/v2/runner.ts`

## Why this matters

This is the seam needed to remove hardcoded runtime behavior.

The calculation flow becomes:

1. Resolve datasets for filing date
2. Normalize case facts
3. Run deterministic means test from bundle
4. Return result + audit metadata

## Current status

The default resolver still returns an embedded bundle built from the current in-repo data. That is transitional.

## Target status

Replace the embedded resolver with a file-backed or database-backed resolver that loads versioned artifacts generated by the official data ingestion pipeline.

## Example

```ts
import { runMeansTestResolved } from "./src/engine/v2/runner";
import { normalizeLegacyInput } from "./src/engine/v2";

const result = await runMeansTestResolved(normalizeLegacyInput(legacyInput));
```
